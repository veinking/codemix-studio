# Seller Package — bIDE by CodeMix

## What the product is

bIDE by CodeMix is a buyer-ready starter SaaS for a browser-based coding and data-science IDE. It combines a polished React frontend, Monaco code editor, multiple runtime adapters, docs/SEO pages, optional Supabase backend, optional AI coding tools, and optional Stripe subscription infrastructure.

## Target customers

- Students learning Python, R, JavaScript, SQL, and other languages.
- Data science learners who need a browser-first coding environment.
- Bootcamps, tutors, and educators needing a customizable IDE starter.
- Founders buying a SaaS starter kit for an online IDE or learning platform.

## Included domains

- Primary: `bideide.com`
- Secondary/redirect candidate: `codemixapp.com` if the seller chooses to include/transfer it separately

## Main features

- Browser-based IDE with Monaco editor.
- Multi-language selection and runtime architecture.
- Python/R/data-science focused positioning.
- CSV/data tooling and visualization-oriented components.
- Docs/tutorial/use-case/SEO content pages.
- Optional Supabase auth, cloud saving, sharing, activity, labs, and feedback.
- Optional AI assistant/function flow.
- Optional Stripe Pro subscription scaffolding.
- PWA assets and sitemap/robots.

## Tech stack

React, TypeScript, Vite, Tailwind CSS, Radix UI, Monaco Editor, Supabase, Supabase Edge Functions, Stripe, Pyodide/WebAssembly-oriented runtime support, sql.js, PWA tooling.

## Current status

Buyer-ready starter SaaS package, not a revenue-generating business. The app installs, lints, builds, runs, and previews. Live backend/payment/AI credentials are intentionally not included.

## Growth opportunities

- Add runtime code-splitting and lazy loading.
- Add classroom/team workspaces.
- Add remote compute containers for long-running jobs.
- Expand curriculum/lab marketplace.
- Add template galleries and dataset packs.
- Add enterprise SSO and admin analytics.

## Known limitations

- Supabase, Stripe, and AI require buyer credentials.
- Some runtime behavior needs manual browser QA after deployment.
- Large production JavaScript chunk warning remains because of browser IDE dependencies such as Monaco, Pyodide/runtime adapters, sql.js-style tools, docs pages, and data tools; lazy loading these areas is a recommended post-sale improvement.
- No customer database, email list, Stripe/payment account transfer, social media accounts, AI key, private email, or service credentials are included unless explicitly transferred separately.

## Assets included in sale

- Source code repository.
- Supabase migrations and Edge Functions.
- Public app icons, PWA manifest, Open Graph preview image, sitemap, and robots file.
- Buyer handoff/setup/QA documentation.

## Buyer responsibilities

- Create a buyer-owned Supabase project.
- Add buyer-owned Stripe credentials, product, price, and webhook.
- Add buyer-owned OpenAI or AI gateway credentials.
- Point `bideide.com` to the production frontend.
- Use `codemixapp.com` only as a secondary/redirect domain if included by the seller.
